## 1. 布局決策

VS–002 採用「單一環圍巨構＋少量垂直支系」的世界布局：

- **主體**：中央服務器巨構，由連續主環、中央 NAS core、四條短向心脊和冠體組成；它是全圖唯一的主尺度地標。
- **垂直支系**：兩座較小的 vertical well，位於主體外圍，提供中距離導航和高度層對照。
- **懸浮支系**：一組東北高層懸浮平台和三座懸浮橋；所有次級區域均接回主環，不形成孤島或第二主城。
- **空間類型**：全部是外部平台、外露結構和空間間隙，不建立可進入室內。
- **光粒起點**：位於主體外圍低層，先看見完整主輪廓，再進入低／中／高導航層。

## 2. 初始尺寸與座標

| 項目 | 初始值 | 說明 |
| --- | ---: | --- |
| 地圖水平邊界 | `x/z ±70` | 隱形限制，不建立可見牆。 |
| 可飛行高度 | `y 0.8–61.5` | 接近 blockout 最高冠體。 |
| 主環基礎半徑 | 約 `47`；連續環中心半徑約 `42` | 形成唯一可讀的主環輪廓。 |
| 主環層級 | 主環中心 `y 16`；上層環 `y 29.5` | 先驗證垂直層次，不做細節。 |
| 中央 NAS core | 約 `y 1–49` | S1 只驗證體量，不做表皮。 |
| 中央冠體 | 約 `y 49–58` | 形成遠距離辨識尖端。 |
| Vertical well A | 約 `(-42, 0, +42)` | 外圍西北側中層支系。 |
| Vertical well B | 約 `(+42, 0, -42)` | 外圍東南側中層支系。 |
| Floating group | 約 `(+50, +38, +48)` | 東北高層懸浮支系。 |
| Floating bridges | 主環 → A／主環 → floating group／主環 → B | 保持次級區域互通，不形成孤島。 |

以上數值是 S1 blockout 的空間假設，不是正式模型的最終尺寸；只有 Gate 1 通過後才可作為 S2 詳細建模的比例基線。

## 3. S1.5 整體結構補全

本輪只補充結構連續性，不新增區域與表層細節：

- 6 根主環基座承重柱，把主環落到外部基座；
- 6 根上下層環間斜向承重構件，消除上層環懸空感；
- 核心塔基座環與下部轉接罩，使核心塔嵌入主體；
- 三座懸浮橋的主環端／次級區域端橋頭平台；
- 不增加第二主城、不增加室內、不進入光效和交互。

## 4. Blender 產物

- 可編輯來源：[vs002-s1-world-blockout.blend](./public/assets/vs002/vs002-s1-world-blockout.blend)
- runtime 檢查輸出：[vs002-s1-world-blockout.glb](./public/assets/vs002/vs002-s1-world-blockout.glb)
- 生成腳本：[vs002_s1_blockout.py](./scripts/vs002_s1_blockout.py)

產物規則：

- 僅使用 Blender primitive 和本地程序材質；
- 生成腳本以 Three.js 的 `X/Y/Z`（Y-up）作為設計坐標，再轉換為 Blender 的 `X/Z/Y`（Z-up）後導出，避免環、塔與垂直井軸向錯位；
- 沒有第三方模型、貼圖或外部資產；
- `WORLD_BLOCKOUT` 供 GLB 導出；`NAV_MARKERS` 僅保留在 `.blend` 中供布局檢查；
- 不包含 proximity／click 交互；
- 不把 S1 blockout 視為正式地標表皮或最終建模精度。

## 5. S1.5 Gate 1.5 待驗收項

- 從主要觀察點看，中央巨構有足夠的地標尺度；
- 光粒可在低層、中層、高層之間飛行，不被次要體量卡死；
- Vertical well 和 floating group 能提供高度方向的導航對照，而不是造成第二主體；
- 主環的空間間隙、承重柱、懸浮橋、橋頭平台和垂直冠體具有可讀的視線秩序；
- 地圖邊界與最高高度合理，不讓玩家在 blockout 階段迷失或撞出世界；
- 需要新增資產的缺口能被具體描述，而不是以「場景還不夠豐富」作為理由。
