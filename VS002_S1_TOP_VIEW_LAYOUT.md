# VS–002 S1 俯視圖布局

狀態：S1 俯視圖布局已通過，S1.5 結構補全已建立；尚未進入 S2 細節建模。

## 1. 設計目的

先只確認平面上的主次關係、相互距離與飛行邊界，不使用 3D 方塊代替設計。俯視圖需要讓人一眼讀出「一個核心地標 + 一個連續主環 + 少量垂直／懸浮支援結構」，而不是多個互相競爭的建築群。

## 2. 平面布局決策

| 元素 | 位置／範圍 | 平面意義 |
| --- | --- | --- |
| 飛行邊界 | `X/Z ±70` | 隱形地圖外框，不做可見牆體 |
| 核心地標 | 原點附近，約 `±10` 平面占地 | 全圖唯一主視覺錨點；高度複雜度留到 S2 |
| 主環巨構 | 以原點為中心，內半徑約 `38`、外半徑約 `47` | 單一連續外部巨構，包覆核心但不形成室內 |
| 垂直井 A | 西北，約 `(-42, +42)` | 第一個次級垂直支點，與主環有單一連接 |
| 垂直井 B | 東南，約 `(+42, -42)` | 第二個次級垂直支點，與 A 形成斜向平衡 |
| 懸浮建築群 | 東北，約 `(+50, +48)` | 唯一懸浮群；保持遠離核心，避免成為第二主城 |
| 懸浮橋 A | 主環西北段 → 垂直井 A | 把西北區接回主環，保持飛行穿越縫隙 |
| 懸浮橋 B | 主環東北段 → 懸浮建築群 | 使懸浮群成為可抵達的支援節點，而非孤島 |
| 懸浮橋 C | 主環東南段 → 垂直井 B | 把東南區接回主環，與橋 A 形成非對稱網絡 |
| 玩家起點 | 西南，約 `(-55, -47)` | 進入後先看到核心與主環的斜向構圖 |

## 3. 明確不放入本階段的內容

- 建築表層皮膚、窗格、管線、標誌與材質細節
- 靠近／遠離觸發點
- 點擊信息節點
- 室內空間
- 第二座主地標或大型城市群

## 4. 懸浮橋網絡原則

- 三個次級區域各自至少有一條懸浮橋接入主環，不形成孤島生態。
- 橋是外部空間的懸浮交通構件，不是室內走廊，也不是地面道路。
- 橋與核心保持可穿越的垂直空隙，不能把中央空間封成環內庭院。
- S1 只確認橋的端點、方向與互通性；橋的高度差、支撐方式、光效與材質留到 S2。

## 5. Gate 1 審查問題

1. 核心地標是否在平面上具有絕對主導位置？
2. 主環是否應保持完整閉合，還是需要一至兩個有意義的開口？
3. 西北與東南垂直井的距離是否足以支持飛行探索？
4. 東北懸浮群是否仍像支援性結構，而不是第二個中心？
5. 西南出生點是否能提供清楚的第一視線？
6. 三座懸浮橋是否讓所有區域互通，同時保留足夠的飛行穿越空間？
