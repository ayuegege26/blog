# VS–002 S2 正式項目：全局細節 Pass

狀態：Gate 2B 外殼、Gate 2C 材質與天空、Gate 2D 主塔表面語言、Gate 3 靠近／離開交互均已完成；目前在同一正式路由進入 S4 编号点击节点验收。

## 1. 範圍

本 pass 建立在已驗收的 S1.5 世界結構上，使用已確認的主塔／垂直井細節語彙擴展到：

- 核心地標塔；
- 西北與東南兩座垂直井；
- 主環上下層的邊緣帶與模組縫；
- 東北懸浮群的平台邊緣、體量嵌入帶；
- 三座懸浮橋的單側邊緣構件。

## 2. 已加入的正式細節

- 核心塔上下段的垂直結構肋；
- 核心塔上下段的連續外部面板；
- 三層低段數結構環帶；
- 四條克制的冷青嵌入帶；
- 核心塔外部橋接平台細節；
- 兩座垂直井的四向結構肋與兩層環帶；
- 兩座垂直井的連續外套筒；
- 主環與上層環的封閉環殼、低段數邊緣帶，以及對齊承重節奏的 6 組維護節點；
- 懸浮群平台收邊、雙建築模組分段、八角懸吊核心與四條短支撐；
- 三座懸浮橋的中央底部骨架、橫向繫桿、高側邊與低側路緣；
- 縮減橋頭轉接盤比例，使其回到結構節點而非視覺主體。

細節仍以低／中模、程序化 Blender 組件為主，沒有室內或高密度 3A 建模；表面貼圖只在主塔核心與橋接節點做有限驗證。

## 3. 材質與天空光照方案

全局材質以共享 runtime token 建立以下層級；主塔核心與橋接節點額外使用共享的低成本顏色／粗糙度表面貼圖：

| 層級 | 視覺角色 | 表現 |
| --- | --- | --- |
| 地面 | 最暗背景基底 | 深藍灰、高粗糙度、低金屬度 |
| 主塔 | 唯一 hero landmark | 中亮冷灰、較高金屬度、清晰受光面 |
| 主環／垂直井 | 次級建築骨架 | 中深藍灰，粗糙度略高於主塔 |
| 橋體／懸浮群 | 交通與外圍模組 | 冷灰藍，與主環保持可辨識差異 |
| 結構肋 | 最深結構線 | 深藍灰、高金屬度、低亮度 |
| 嵌入帶／環帶 | 克制結構光 | 低強度冷青 emissive，不照亮整張地圖 |

天空使用無外部貼圖的程序漸層：高空深藍、地平線暮光與低透明度薄雲帶作頁面最底層，透明 3D 畫布疊加其上，因此沒有天空球邊界、貼圖接縫或 renderer 排序差異。Hemisphere Light 作主要中量光源，方向光只補充結構陰影；霧、ACES tone mapping 和曝光只服務遠近層級。

表面貼圖目前只套用於 `MAIN_SERVER_CORE` 與 `S2_CORE_DOCK_DETAIL`，用來確認面板分區、粗糙度變化和冷青維護標記能否把結構胚體推進到科技建築表皮。通過 Gate 2D 後，再按主環、垂直井與懸浮橋的實際需要逐區擴展。

## 5. Gate 2C / Gate 2D 驗收

1. 天空是否完整覆蓋所有飛行位置，不再出現固定球體邊界？
2. 天空是否能作為主要中量光源，使暗部建築仍可讀？
3. 主塔、主環、垂直井、橋體與懸浮群是否具有清楚且克制的材質層級？
4. 冷青嵌入光是否只強調結構，不搶奪主塔視覺權重？
5. 遠景、中景與近景是否都沒有過曝、死黑或霧化失焦？
6. 畫面是否仍保持冷淡、科技、神秘，而不是 3A 寫實或卡通小鎮？
7. 主塔表面貼圖是否只增加面板、粗糙度與維護節奏，而沒有變成規則棋盤或卡通紋理？
8. 主塔貼圖是否在近景可辨識、中景不喧賓奪主、遠景不形成噪點？

Gate 2D 通過後凍結 S2.5 表面語言。S3 使用獨立 proximity registry 驅動主塔、上下環、兩座垂直井、懸浮群與三座橋體的局部結構蘇醒；進入與離開使用不同閾值，離開後平滑復原，且不加入點擊內容。S3 不得改動已凍結的建築外殼材質與表面貼圖，只能讓既有燈帶、嵌件和少量結構肋分段提升發光，並保留極低幅度的慢呼吸。

Gate 3 通過後冻结 S3 距离、渐变和复原状态机。S4 在同一路由加入 01–09 九个粒子节点；只复用 probe 的靠近显现、点击展开、再次点击／`Esc` 收起语法，展开内容暂用 placeholder，正式文案延后到收口阶段填写。
