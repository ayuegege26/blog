# VS–001 Archive Atlas／档案地图

> 状态：第二阶段真实 Archive 数据接入已完成，当前由七个公开 MDX 生成。
>
> 定位：博客内容探索工具，不是生成式壁纸、关系图谱演示或 Archive 列表的视觉复制。

## 1. 项目目的

Archive Atlas 为 Ayue Observatory 提供第二种内容入口：普通 Archive 适合按列表查阅，Atlas 负责让访问者观察内容如何随时间、类型和主题逐渐积累，并从可视空间直接进入文章。

它必须同时满足两个条件：

1. 对访问者有实际用途，可以发现、筛选并打开博客内容。
2. 使用足够扎实的前端技术处理真实内容规模、连续交互、响应式布局与可访问性，而不是用技术名词装饰画面。

## 2. 数据来源

### 2.1 两阶段交付

第一阶段先完成完整项目结构，使用描述性占位资料而不是伪造内容：

- 每个占位字段直接说明此处应填入的资料类型，例如“文章标题”“首次发布时间”“显式关联对象”。
- 搜索、筛选、Canvas 时间空间、节点选择、显式关联、URL 状态、键盘等价列表和移动端布局全部可运行。
- 文章入口保持禁用，并明确标注将在第二阶段替换；第一阶段不得制造看似真实的标题、日期或阅读数据。
- 第一阶段公开路由为 `/lab/visual-systems/archive-atlas/`，状态为 `prototype`，不标记为完成品。

第二阶段只替换资料来源和入口，不推翻第一阶段的交互结构：

- 从公开 Archive Objects 生成索引，将占位字段映射为真实字段。
- 启用每个对象的公开文章 URL，并用真实时间、类型、状态、标签和阅读长度重新计算视觉映射。
- 逐项校验缺失字段、空结果、异常长度和显式 `related`，不得用推测关系补齐资料。
- 真实资料接入、内容抽查、性能与桌面／移动 QA 全部通过后，才把项目状态改为 `available`。

### 2.2 第二阶段真实数据边界

所有数据在 Astro 构建阶段从公开 Archive Objects 生成，不调用私人服务，也不把草稿送到浏览器。

首版字段：

- `objectId`、`slug`、标题和摘要。
- 发布时间与最近更新时间。
- 内容类型、状态与公开标签。
- 字数和估算阅读时间。
- `featured`、`priority` 和明确填写的 `related`。
- 对应公开文章 URL。

禁止根据模糊文本相似度自动声称文章存在关系。首版关系线只来自显式 `related`；共同标签只用于筛选和聚类提示，不等同于依赖关系。

## 3. 视觉映射

- 横轴：发布时间，保持真实时间比例；可在年份、月份和局部日期之间缩放。
- 纵向区域：内容类型或主板块，不表达价值高低。
- 节点大小：字数或阅读时间，提供图例并限制极端值。
- 节点色彩：内容类型；状态使用边框或纹理辅助表达，不能只靠颜色。
- 更新时间：从原始发布时间延伸一条克制的短轨迹。
- 显式关联：选中节点后才显示，避免默认画成杂乱网络。

原 Temporal Field 的流场或 Shader 只能作为低对比度背景响应：例如缩放、选中或内容密度变化时提供轻微空间反馈。关闭该层后，Atlas 的信息和操作必须完全成立。

## 4. 核心交互

- 缩放和拖动时间空间。
- 按类型、年份、标签和状态筛选。
- 搜索标题与摘要。
- 选择节点后显示标题、摘要、日期、阅读时间和进入文章按钮。
- 在选中状态查看显式关联对象。
- 将筛选、缩放区间和选中对象写入 URL，允许刷新和分享。
- 提供“回到全部内容”和“回到最新内容”。

移动端不照搬桌面画布：默认使用纵向时间带与底部详情面板，保留搜索、筛选和文章跳转。

## 5. 技术设计

- Astro 构建阶段生成最小公开索引 JSON。
- Web Worker 完成搜索索引、筛选、布局输入和统计聚合，避免阻塞主线程。
- Canvas 2D 作为首版正式渲染器；当内容规模确实需要时再引入 WebGL，不为了技术展示提前增加复杂度。
- 使用空间索引处理节点命中测试和视口裁剪。
- 只渲染视口附近节点，并限制高 DPI 像素比例。
- DOM 详情面板承担正文和操作，Canvas 不独占信息语义。
- 提供与画布同步的键盘可访问结果列表。
- 尊重 `prefers-reduced-motion`，后台和离页时停止绘制并释放监听器。

## 6. 页面结构

```text

├── 搜索／筛选／时间范围

```

## 7. 分阶段验收

### 7.1 第一阶段结构原型

- 页面和 Lab／Visual Systems／主站入口可访问，并诚实显示 `STAGE 1 / PLACEHOLDER DATA`。
- 占位节点只描述字段类型，不伪装成真实博客内容。
- 搜索、类型／状态／时间筛选、重置、缩放、拖动、选择、关联线和 URL 状态均可运行。
- 桌面横向时间轴、移动端纵向时间轴和可访问结果列表均可使用。
- Canvas 与 Worker 只在项目路由加载，离页时释放监听和 Worker。

### 7.2 第二阶段真实内容

- 使用真实公开 Archive 数据，不使用为了画面好看而伪造的节点。
- 可以从 Atlas 找到并进入任意公开对象。
- 所有视觉映射有图例且不会暗示不存在的语义。
- 桌面鼠标、键盘和移动触控均可完成搜索、筛选、选择和跳转。
- 刷新或分享 URL 后可以恢复视图状态。
- 关闭动画、Canvas 不可用或启用 reduced-motion 时仍可通过列表使用核心功能。
- 200、1,000 个模拟公开节点下保持交互可用，并记录性能结果。
- Observatory 主站不加载 Atlas 的 Worker、索引或图形代码。

以上第二阶段验收已于 2026-07-16 完成；内容核查记录见 `ARCHIVE_ATLAS_CONTENT_AUDIT.md`。
