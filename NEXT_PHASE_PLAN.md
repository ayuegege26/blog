# Ayue Observatory Next Phase Plan

## 目的

本文档用于 `v0.1.2` 之后的下一阶段讨论。

当前首批 OBS 对象已经基本成形，站点不再只是结构样板，而是有了一组可以互相支撑的档案对象。下一阶段的重点不应继续盲目扩充对象数量，而是让现有对象作为一个可阅读、可维护、可发布的系统成立。

---

## 2. 下一阶段原则

### 2.1 不急着增加新对象

除非出现非常明确的新内容，否则下一阶段不以“新增对象数量”为目标。

首批对象已经能表达站点的基本边界。继续加内容之前，应该先确认这些对象在页面上的阅读顺序、相关对象关系和栏目归属是否稳定。

### 2.2 不把站点改成 dashboard

系统、工具、自动化和数据内容可以继续作为 Archive Object 出现，但它们不应反过来定义首页和站点身份。

首页仍然应是 Observatory 入口，而不是 NAS 状态页、金融面板或工具导航页。

### 2.3 先做阅读体验，再做新功能

下一阶段的主要工作应该是：

- 检查页面文案是否和首批对象一致。
- 调整首页与 Archive 的展示节奏。
- 优化对象详情页的阅读体验。
- 校准相关对象的跳转关系。
- 完成一次真实浏览 QA。

## 3. 建议工作包

### Work Package A: 首页与 Archive 校准

目标：让首页第一屏和 Archive 总索引准确反映当前对象矩阵。

建议检查：

- 首页 `Observation Log` 是否仍然准确。
- `Archive Snapshot` 是否应该按 OBS-0001 到 OBS-0008 的逻辑展示。
- `featured` 和 `priority` 是否与当前阅读顺序一致。
- Archive 页是否需要更明确地区分“首批 OBS 对象”和后续隐藏对象。

不建议：

- 新增复杂筛选。
- 新增搜索。
- 把首页改成对象 feed。

### Work Package B: 栏目页文案校准

目标：Projects / Essays / About 的说明文案与新对象关系保持一致。

建议检查：

- Projects 页是否能准确承载 OBS-0005。
- Essays 页是否应解释 OBS-X 是隐藏对象，而不是主阅读列表核心。
- About 页是否需要补一句“未知 / 未来”对象的存在意义。

不建议：

- 新增一级栏目。
- 把 OBS-X 做成显眼入口。
- 为 `unknown` 单独开栏目。

### Work Package C: 对象详情页阅读 QA

目标：确认 15 个静态页面都能自然阅读。

检查路径：

- `/`
- `/archive/`
- `/projects/`
- `/essays/`
- `/about/`
- `/archive/digital-space-essay/`
- `/archive/linear-algebra-notes/`
- `/archive/self-hosted-layers/`
- `/archive/market-signal-format/`
- `/archive/nas-agent-project/`
- `/archive/container-network-failure/`
- `/archive/object-card-motion/`
- `/archive/june-observation-journal/`
- `/archive/obs-x/`

重点看：

- 标题是否过长。
- 标签与状态是否自然。
- 相关对象是否构成有意义路径。
- 移动端是否出现明显拥挤。
- 长文段落是否太密。

### Work Package D: 发布与版本节奏

目标：形成稳定的小版本发布方式。

建议：

- `v0.1.2`: 首批 OBS 对象矩阵完成。
- `v0.1.3`: 首页 / Archive / 栏目页校准。
- `v0.1.4`: 阅读体验、移动端和细节 QA。
- `v0.2.0`: 只有在确认需要新结构时再进入，例如对象关系页、专题集合或搜索。

## 4. 下一步讨论问题

下一阶段开始前，建议先讨论这几个问题：

1. 首页应该更强调 OBS-0001，还是按当前更新时间展示对象？
2. OBS-X 是否只作为 OBS-0008 的相关对象存在，还是允许在 Essays 页被自然看到？
3. `unknown / future` 是只给 OBS-0008 使用，还是未来允许更多类似对象？
4. Archive 页是否要加入一段简短说明，解释首批 OBS 对象矩阵？
5. 下一轮最优先校准的是首页、Archive，还是 About？

---

## 5. 推荐方向

推荐下一阶段先做 **首页与 Archive 校准**。

原因：

- OBS 对象已经完成，但访客第一次看到的仍然是首页和 Archive。
- 首页当前文案还来自较早阶段，未完全吸收 OBS-0008 和 OBS-X 的变化。
- Archive 是全部对象的总索引，最适合作为对象矩阵的稳定入口。

下一阶段如果只做一件事，就应该让首页和 Archive 变成当前内容系统的准确入口。
