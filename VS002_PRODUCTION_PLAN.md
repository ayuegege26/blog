## 0. 生產總規則

VS–002 不採用一次成型流程。每一階段完成後，必須通過本文件的驗收門，才能進入下一階段；未通過時只修正當前階段，不向後疊加內容。

### 0.1 階段順序

```text
S1 地圖佈局與建模
  ↓ Gate 1 / 空間與尺度合格
S2 主體細節建模與全局／局部光效
  ↓ Gate 2 / 視覺語彙與資產品質合格
S3 靠近／遠離觸發系統
  ↓ Gate 3 / 可逆狀態與密度合格
S4 點擊節點與內容協商
  ↓ Gate 4 / 信息、交互和可訪問性合格
S5 收口與 Lab／主題項目銜接
  ↓ Gate 5 / 發布候選合格
```

### 0.2 不確定項目處理規則

以下任一情況都不得直接在正式版中試錯：

- 不確定某種鏡頭、飛行、碰撞、光效或 shader 表現是否成立；
- 不確定某種資產包是否能融入當前風格或性能預算；
- 不確定某種靠近／點擊交互是否能被理解；
- 不確定信息面板、世界內文字或粒子提示的布局；
- 不確定與 Lab／主題項目的入口、返回和內容映射方式。

處理流程固定為：

1. 在正式項目之外建立最小化實驗或臨時測試分支。
2. 為實驗寫明「要驗證的假設、最小輸入、成功條件、失敗條件」。
3. 實驗通過後，記錄決策、參數和已知限制。
4. 只有通過的能力才可轉入正式項目；失敗實驗的代碼和資產不得被正式場景隱式依賴。

### 0.3 共同產物規則

每一階段都必須更新：

- 階段決策記錄；
- 變更後的場景／資產 manifest；
- 已知限制和未決問題；
- 當前可回退的穩定版本。

任何新增素材都必須說明：它解決哪個已觀察缺口、為何既有資產不能改造、預計使用數量、runtime 成本和授權狀態。

## S1. 規劃地圖整體佈局，然後建模

### 目標

先確定世界的空間秩序，再建立可飛行、可觀察、可測試的低模地圖。這一階段不做正式表層皮膚、不做完整交互、不填寫最終內容。

### 必須先確定

- 單一環圍巨構的主體位置、朝向、比例和遠景輪廓；
- 少量垂直井城／懸浮建築群的數量、層級和與主體的關係；
- 低層、中層、高層導航路徑；
- 玩家起始點、主要觀察點、飛行邊界和最高海拔；
- 3–5 個 hero landmark 的暫定位置和空間功能；
- 外部平台、外露通道、懸空結構和空白區的比例；
- NAS 主體與其他計算機工具在空間上的初始分區方式。

### 生產內容

1. 先製作 2D 俯視／側視布局圖和高度分層圖。
2. 用低模 primitive、基礎模組和簡化碰撞 proxy 建立 Blender blockout。
3. 導出最小 GLB，接入正式 runtime 的相機、光粒和邊界。
4. 用固定測試路徑檢查起點到各高度層的飛行時間、視線可讀性和空間迷失風險。

### Gate 1：空間與尺度驗收

- 不看信息文字，也能辨認主巨構、支系和主要地標的層級；
- 第三人稱光粒能完成低／中／高三層飛行；
- 細小結構不會造成卡死，主要建築碰撞能停止並滑動；
- 地圖邊界和最高海拔限制清楚但不破壞視覺；
- 從主要觀察點看，主體不會因比例過小而失去地標性；
- blockout 的幾何與初始 runtime 成本在可接受範圍內。

Gate 1 未通過時，只能調整布局、比例、路徑和 blockout；不得開始 S2 的細節建模。

### S1.5：整體結構補全

當 Gate 1 的平面布局與主體比例獲得確認，但 blockout 顯示承重、轉接或交通連續性不足時，先執行 S1.5，不進入 S2。S1.5 只補充整體結構，不改變已確認的區域數量與平面關係。

- 主環與基座之間的承重柱／支撐；
- 上下層環之間的少量承重構件；
- 核心塔基座與主體的轉接段；
- 懸浮橋兩端的橋頭平台與結構節點；
- 重新檢查低／中／高層飛行時的視線與通行連續性。

#### Gate 1.5：結構連續性驗收

- 上層環不再呈現無理由懸空；
- 核心塔、主環、基座之間有可讀的承重關係；
- 每座懸浮橋都有清晰的起點、落點和轉接平台；
- 不新增第二主城、不加入室內、不偷渡表皮細節；
- 通過後才可進入 S2。

### S1 產出

- `world-layout`：俯視、側視、高度與邊界圖；
- `blockout.blend` 與對應 GLB；
- `navigation-notes`：起點、觀察點、飛行路徑和碰撞規則；
- Gate 1 驗收記錄。

## S2. 建模敲定後，做主體細節建模和光效

### 目標

在 S1 的空間不再變動後，為主體建築、垂直支系和 hero landmark 補充正常低／中模建模精度、材質分層和光效。此階段仍不加入靠近／點擊內容邏輯。

### S2 初始子步驟：核心細節 Pass

正式 S2 先以核心地標塔和一座垂直井作為樣本，驗證垂直肋、低段數環帶、嵌入光帶和橋接平台的細節語彙；樣本通過後，才在同一個正式 pass 內擴展到另一座垂直井、主環、懸浮群與橋體，不另開平行測試項目。

### S2 外殼實體化 Pass

當細節語彙通過初步驗收後，先補足主體的外部建築皮膚，再進入材質與光效。這一 pass 不改變 S1.5 布局與飛行空間，只把「結構骨架」收斂為可從多角度讀出的完整體量：

- 主塔增加連續外部面板；
- 主環保留中心空腔，但使用封閉的箱形／多面體環殼；
- 垂直井增加連續外套筒與基座銜接；
- 橋體增加側壁、底部封板與轉接層；
- 懸浮群補足平台底部、建築外殼和懸吊支撐。

外殼 Pass 通過前，不開始 S2 材質／光效收斂；仍使用同一正式 S2 資產，不另開測試入口。

### S2 材質與藍調天空光照 Pass

外殼 Pass 通過後，仍在同一正式路由內按以下順序收斂：

1. 先建立地面、主塔、主環、垂直井、橋體、懸浮群、結構肋與發光嵌入件的共享材質 token；
2. 以藍調時刻天空和 Hemisphere Light 作主要中量光源，方向光只補充結構陰影；
3. 以無外部貼圖的程序漸層作天空視覺底層，3D 畫布透明疊加，避免天空球邊緣、接縫與不同 renderer 的排序差異；
4. 使用低密度藍灰霧和 ACES tone mapping 統一遠近層級；
5. 只保留低強度冷青結構發光，不加入 proximity 或 click 狀態；
6. 遠景、中景、近景依次驗收主塔辨識度、環體連續性、橋體輪廓和暗部可讀性。

本 pass 通過後才進入 S3；若材質或光線不合格，只修改共享 token、天空、霧與曝光，不回改已凍結的建築布局。

### 建模標準

- 正常低／中模建模，不採用 3A 高精度資產；
- 拓撲、法線、bevel、命名、origin 和模組比例可控；
- 無貼圖為預設，先用程序材質和少量可控 decal／atlas；
- 每個主體提供 runtime 幾何、碰撞 proxy 和必要 LOD；
- 第三方資產只能作為已記錄來源的結構零件，必須經比例、材質、命名和性能處理；
- 不因追求表面細節而改變 S1 已驗收的主輪廓。

### 光效分層

1. **全局環境層**：藍調時刻天空作主要光源，為整張地圖提供中量環境光和藍灰遠景。
2. **主體結構層**：建築邊緣、環帶、支撐、平台和冠體的低強度冷色發光。
3. **局部節點層**：只給後續交互節點預留可被狀態驅動的 emissive、粒子和點光源接口。

### Gate 2：視覺與資產品質驗收

- 主體在天空環境光下可被完整讀出，不依賴光粒照亮整張地圖；
- 冷色、材質粗糙度、發光強度和霧的層級一致；
- 建模細節提升了結構辨識度，而不是變成裝飾噪音；
- 所有進入 runtime 的模型都有 manifest、碰撞 proxy、LOD 或明確不需要 LOD 的理由；
- 低品質級別仍能保留主體輪廓和必要發光；
- 未使用未驗證素材或未記錄授權來源。

### S2 產出

- 正式主體與地標 `.blend`；
- runtime GLB／glTF、碰撞 proxy、LOD；
- 材質與光效 token 表；
- 資產 manifest 和 Gate 2 記錄。

## S3. 設計全局靠近／遠離觸發交互

### 目標

先完成所有 proximity 交互的規則和狀態，不放入點擊信息內容。靠近／遠離只負責空間反應和可感知的狀態變化。

### 生產內容

1. 為每個 proximity 節點建立穩定 `id`、類型、範圍、優先級、可用條件和狀態。
2. 建立全局 proximity registry，統一處理進入、持續、離開和禁用。
3. 為每個節點指定局部蘇醒序列：優先使用模型既有燈帶、嵌件與少量結構肋，不改建築外殼材質。
4. 驗證進入與離開的可逆性；離開是復原條件，不以一次性動畫播放為核心。
5. 建立不同距離、同時多節點和高速飛行下的優先級規則。

### S3 正式觸發表

| 節點 | 形狀 | 進入／離開遲滯 | 優先級 | 局部反應 |
| --- | --- | --- | --- | --- |
| 核心主塔 | 垂直圓柱體積 | 5.5／9 | 100 | 嵌入帶、燈帶與少量結構肋由低至高分段蘇醒 |
| 上層環 | 環形體積 | 4／7 | 90 | 既有邊緣燈帶與節點依次蘇醒，環體外殼不變 |
| 主環 | 環形體積 | 4.5／7.5 | 86 | 維護節點、燈帶與少量支撐形成低幅度蘇醒 |
| 垂直井 A／B | 垂直圓柱體積 | 5／8 | 80 | 井體燈帶與嵌件分段蘇醒，套筒外殼不變 |
| 懸浮群 | 球形包圍體 | 6／10 | 76 | 建築嵌件與少量懸吊結構分段蘇醒 |
| 懸浮橋 A／B／C | 線段膠囊體積 | 4／7 | 70 | 高側燈帶、底部節點沿橋向柔和蘇醒 |

所有節點使用同一 registry 更新，進入距離小於離開距離，避免在邊界反覆閃爍。多節點可以同時保有 activation，但 HUD 與極弱的局部光溢出只跟隨優先級最高的 focus target。`prefers-reduced-motion` 下停用呼吸，只保留分段狀態漸變。

S3 的視覺契約是「結構蘇醒」，不是「選中高亮」：不得改動建築外殼的底色、材質 token、表面貼圖或整體亮度，不得以整體青色覆蓋表達進入狀態。反應只落在模型預先設計的發光通道，按照語義順序柔和進入；穩定狀態只允許低幅度慢呼吸，離開時按同一 activation 平滑回到原始值。

### Gate 3：交互狀態驗收

- 每個節點進入和離開都能穩定觸發；
- 蘇醒序列在中近景可辨識、近景不過曝，且不造成整體建築高亮；
- 外殼底色、表面貼圖與非發光結構在進入前後保持一致；
- 離開後能平滑恢復，重複進入不累積錯誤狀態；
- 同時接近多個節點時，主要 focus target 有穩定規則；
- reduced-motion、低品質和 WebGL fallback 仍能辨認狀態；
- proximity 交互不偷偷承擔點擊信息或敘事進程。

### S3 產出

- proximity registry 與狀態表；
- 全局觸發距離和優先級規則；
- 每個節點的光效配置；
- Gate 3 交互驗收記錄。

## S4. 設計點擊交互點，使用占位內容驗證

### 目標

在 proximity 狀態穩定後，建立所有 click node，使用占位內容完成點擊命中、信息展開、切換、收起和降級路徑驗證。正式節點內容不在 S4 填入，統一留到 S5 收口時，作為 VS–002 與博客主項目的內容銜接工作。

### 生產內容

1. 列出所有 click node 的 `id`、位置、所屬主體、代表的 NAS／工具／博客內容和優先級。
2. 先用 placeholder 文字驗證世界內信息表現，不提前寫最終文案。
3. 驗證每個節點的世界內粒子、信息面板和 DOM fallback 都能使用同一個選中狀態。
4. 驗證節點切換、任意位置收起、ESC、離開區域和 WebGPU／WebGL／HTML fallback。

### S4 编号节点表

正式地图使用 9 个节点，数量冻结在十个以内；每个节点引用一个明确的 proximity 区域，上层环的四个新节点使用独立方位分区，不另建测试路由。

| 编号 | `id` | 所属区域 | 当前内容状态 |
| --- | --- | --- | --- |
| 01 | `node-01-core` | 核心主塔顶部 | placeholder |
| 02 | `node-02-upper-ring` | 上层环 | placeholder |
| 03 | `node-03-main-ring` | 上层环／东段 | placeholder |
| 04 | `node-04-well-a` | 垂直井 A 顶端 | placeholder |
| 05 | `node-05-well-b` | 垂直井 B 顶端 | placeholder |
| 06 | `node-06-floating-group` | 悬浮建筑群 | placeholder |
| 07 | `node-07-upper-ring-north` | 上层环／北段 | placeholder |
| 08 | `node-08-upper-ring-west` | 上层环／西段 | placeholder |
| 09 | `node-09-upper-ring-south` | 上层环／南段 | placeholder |

节点复用 probe 已通过的视觉语法，但远距状态不因交互状态机而完全隐藏：每个节点保留一个体积小、低强度的冷蓝紫信标，同时必须遵守场景深度与建筑遮挡，不允许透视实体；进入范围后才渐进展开实体核心、双轨道、线框场和少量伴随粒子。交互节点不得使用操控光粒的白色／冷青主色和同等强度。点击后增强节点自身的场与轨道并展开邻近信息面板，不改变建筑外壳材质。再次点击同一节点、点击任意空白位置或按 `Esc` 均可收起；点击另一节点时直接切换；离开当前节点的退出距离后信息面板必须自动收起。正式内容在 S5 使用 [VS002_S4_NODE_CONTENT_TEMPLATE.md](./VS002_S4_NODE_CONTENT_TEMPLATE.md) 填写。

S4 不建立第二套距离状态机：九个 click node 必须引用 S3 proximity registry 的区域 `id`、进入／退出状态与 activation；同一结构需要多个点击节点时，在 registry 中增加独立方位分区，确保“结构苏醒”先于或同步于“节点可点击”。视觉深度遮挡与命中遮挡必须一致，射线命中前先检查正式建筑实体，禁止点击被建筑遮住的节点。节点离开镜头正面、可视范围或被实体遮住时，已展开的信息自动收起。粒子呼吸、轨道与伴随粒子旋转都必须遵守 `prefers-reduced-motion`；键盘路径以与世界节点投影对应的原生 DOM 控件提供，并与 pointer 使用同一选中状态和信息面板。

### Gate 4：點擊交互驗收

- 九個節點都有穩定的 `id`、S3 proximity 對應關係和占位內容；
- 點擊命中、信息開關、節點切換和重置規則清楚；
- 世界內粒子表現與文字信息互相支持，不依賴傳統屏幕按鈕作唯一入口；
- 鍵盤、pointer、DOM fallback 和 reduced-motion 路徑一致；
- 點擊信息不會錯誤清除 proximity 的可逆光效狀態。

### S4 產出

- click registry；
- 九個節點的占位信息表現；
- 世界內信息表現與 DOM fallback；
- Gate 4 記錄。

## S5. 整體收口，銜接 Lab–VS–002 與主題項目

### 目標

把已驗收的世界、交互和內容整理成可部署的 VS–002 板塊，並確定它如何在 Lab 和博客主題項目中被理解、進入、返回和引用。此階段不再添加未驗證的新能力。

### 收口內容

- WebGPU、WebGL fallback、HTML fallback 和載入失敗狀態完整；

### S5 输入与浏览器会话韧性优化步骤

1. 操控输入统一读取 `KeyboardEvent.code`，不再依赖受中文输入法、大小写锁定和键盘布局影响的 `event.key`。
2. 窗口失焦、页面隐藏、指针捕获丢失和离页时必须立即清空所有移动状态与残余速度，避免漏收 `keyup` 后持续移动。
3. `Ctrl` 不再承担飞行控制：浏览器保留 `Ctrl+W` 关闭标签，网页无法可靠覆盖。正式键位改为 `WASD` 平移、`Shift` 上升、`C` 下降。
4. 点击 3D 画布时主动恢复画布焦点；键盘监听使用捕获阶段，但不得干扰页面外的表单和系统快捷键。
5. S5 稳定性验收默认使用 WebGL；WebGPU 暂以 `?renderer=webgpu` 显式启用，完成连续运行、切换标签、输入法切换和显卡驱动回归后再决定是否恢复默认。
6. 完成至少 20 分钟连续移动测试，并覆盖中文输入法、Caps Lock 开关、快速切换标签、窗口失焦、同时按住两个移动键及松开修饰键的场景；不得出现粘滞、单键失效或标签被控制键组合关闭。

## 6. 階段回退與變更控制

任何跨階段改動都必須在變更記錄中說明：觸發原因、受影響階段、回退點、重新驗收項和是否需要建立新的最小化實驗。
